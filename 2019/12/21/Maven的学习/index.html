Maven的学习

发表于 2019-12-21 更新于 2019-12-28

1.执行与Maven构建相关的命令时，必须进入pom.xml所在的目录下

与构建过程相关：编译、测试、打包

构建过程中的各个环节：

1. 清理：将以前编译得到的旧的class字节码文件删除，为下一次编译做准备
2. 编译：将Java源文件编成字节码文件
3. 测试：自动测试，调用junit程序
4. 报告：测试程序执行的结果
5. 打包：动态Web工程打war包，Java工程打jar包
6. 安装：Maven特定的概念————将打包得到的文件复制到仓库中指定位置
7. 部署：将动态Web工程生成的war包复制到Servlet容器的指定目录下，使其可以运行

2.常用命令

mvn clean 清理
mvn compile 编译主程序
mvn test-compile 编译测试程序
mvn test 执行测试   
mvn package 打包
mvn install 安装

执行Maven用到某些插件时会先去本地仓库找

Maven本地仓库的位置:

C:\Users\登录当前系统的用户名.m2\repository

 
Maven核心程序如果在本地仓库中找不到所要的插件，便会自动链接外网,中央仓库下载

修改本地仓库的位置

 
在D:\maven\apache-maven-3.6.3\conf\settings.xml文件中，如下标签中修改仓库位置

<localRepository>D:\repository</localRepository>

3.Maven工程的坐标

使用三个向量在仓库中唯一定位一个Maven工程

  • groupId：公司或组织域名+项目名
  • artifactId：模块名
  • version：版本

4.依赖

Maven解析依赖信息时会从本地仓库查找被依赖的jar包

1）对于我们自己开发的Maven工程，使用mvn install命令安装后就可以进入仓库
比如：A项目中，依赖了B项目

<!--依赖B工程-->
</dependencies>
  <dependency>
      <groupId>com.haida.mvnbook</groupId>
      <artifactId>B</artifactId>
      <version>1.0-SNAPSHOT</version>
  </dependency>
</dependencies>

那么就需要到B项目下执行mvn install命令，这样在本地仓库中就有B项目的相关文件了，A项目才可以正常引用B项目

5.在eclipse中创建Maven工程

new一个Maven的java工程，勾选上创建一个简单的Maven（没有骨架）如图

点Next，下一步，填写GroupId和ArtifactId

这里会有问题，自动生成的java的版本是1.5的，一般用的是1.8的或更高

 

有两种方法解决：

(1)第一种，每次都要改。

 
对项目右击，Properties > Java Build Path > Libraries

还要再修改Java Compiler

这样就改好了

(2)第二种，一劳永逸。

1)打开maven的conf文件夹下settings.xml文件

2)找到profiles标签，在其中添加下面的标签

<profile>
    <id>jdk-1.8</id>
    <activation>
        <activeByDefault>true</activeByDefault>
        <jdk>1.8</jdk>
    </activation>
    <properties>
        <maven.compiler.source>1.8</maven.compiler.source>
        <maven.compiler.target>1.8</maven.compiler.target>
        <maven.compiler.compilerVersion>1.8</maven.compiler.compilerVersion>
    </properties>
</profile>

再新建一个Maven工程，结果就是1.8的版本了
